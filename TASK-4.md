# Penetration Testing Methodology

A structured penetration testing approach was studied throughout this task.

## Phases

### 1. Reconnaissance

Collected basic information about the target system.

### 2. Scanning

Identified available services, open ports, and potential attack surfaces.

### 3. Exploitation

Studied how known vulnerabilities may be responsibly tested in an isolated and authorized lab environment.

### 4. Post-Exploitation

Learned how security professionals gather information after obtaining authorized access to understand the overall security posture of a system.

### 5. Reporting

Documented observations, potential risks, and recommended mitigation strategies.

Password auditing concepts were studied to understand the importance of strong authentication.

Topics Covered:

- Password complexity
- Brute-force attack concepts
- Dictionary attacks
- Password hash concepts

Tools Studied:

- Hydra
- John the Ripper

Learning Outcome:

---

# Social Engineering Awareness

Studied common phishing techniques and methods used by attackers to deceive users.

Topics Covered:

- Email phishing
- Fake login pages
- Credential theft awareness
- User awareness training

Best Practices:

- Verify sender identity
- Check URLs carefully
- Avoid opening suspicious attachments
- Enable Multi-Factor Authentication (MFA)

# System Hardening

System hardening reduces the attack surface and improves overall security.

Security Measures Studied:

- Install operating system updates
- Apply security patches
- Configure firewall rules
- Disable unused services
- Remove unnecessary software
- Use strong passwords
- Enable logging and monitoring
- Apply least privilege principles
